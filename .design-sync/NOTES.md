# design-sync NOTES — ShiroAni

Target project: `7176d6a1-b9b3-4e0f-bc28-276bbfa842a5` (ShiroAni Design System).

## Repo shape

- Source is `apps/web` — the web **application** (`@shiroani/web`), NOT a published component library. No `dist/`, no barrel exporting `window.<Global>.*`. A barrel entry is authored for the converter `--entry`.
- Storybook: `@storybook/react-vite` v10, config at `apps/web/.storybook`. Tailwind v4 (`@tailwindcss/vite`), themes via class names (`.storybook/preview.tsx` uses `withThemeByClassName` from `@storybook/addon-themes`).
- React 19.2.
- 179 storied components across 13 feature groups + `ui/` primitives. Full-scope sync requested.

## Story render dependencies (from .storybook/preview.tsx)

- `TooltipProvider` (Radix) wraps every story.
- `I18nextProvider` with the real `@/lib/i18n` instance (18 namespaces); language driven by toolbar global, default `DEFAULT_LANGUAGE`.
- A dormant socket is initialized (`initializeSocket(0)`, autoConnect false) so view hooks' socket listeners don't throw.
- Theme classes applied to `:root` (dark themes also get bare `dark` class for Tailwind `dark:`).
- Stories seed Zustand stores directly; never hit the backend.

## Global learnings (flushed in solo phase — apply everywhere)

- [GENERAL] App-shaped repo: no library dist. A barrel `apps/web/.ds-entry/index.tsx` is generated by `.design-sync/gen-entry.mjs` (run via `buildCmd`) from `sb-reference/index.json`. It re-exports every storied component under its story-title name. EVERY export routes through `import {x as _X}/_X default → export const X = _X` because many components are `export default memo(...)` (ts-morph won't count the ExportAssignment as a value decl otherwise). It also `export *`s each component module so co-located sub-components (e.g. SettingsToggleRow) land on the global — story-imports shims the whole module to the global when a story imports a sibling from it.
- [GENERAL] Decorators can't bundle (Tailwind v4 `@import "tailwindcss"` in preview.css). Replaced by `cfg.provider = DesignProvider` (hand-authored `apps/web/.ds-entry/DesignProvider.tsx`): adds `plum dark` to `<html>` (theme tokens are scoped `:root.<theme>`; plum is the default dark theme), wraps in I18nextProvider + TooltipProvider, inits a dormant socket, AND sets `document.body` background/color to `var(--background)`/`var(--foreground)` (the preview card hardcodes a white body; without this, dark-theme light text is invisible).
- [GENERAL] CSS comes from `[CSS_FROM_STORYBOOK]` (the compiled Tailwind in `sb-reference/assets/iframe-*.css`). `fallbackCssFromStorybook` picks the LARGEST local `<link rel=stylesheet>` in iframe.html — do NOT inject a competing `<link>` to fonts.css (it's bigger and wins, leaving `_ds_bundle.css` fonts-only and every preview unstyled). Fonts are injected into the reference via an inline `<style>@import "./fonts/fonts.css"</style>` (neither scraper reads `<style>`). On any sb-reference rebuild, re-copy `ds-bundle/fonts`→`sb-reference/fonts` and re-inject that `<style>` line (use `data-ds-injected-fonts` marker).
- [GENERAL] Fonts: app imports `@fontsource-variable/{dm-sans,jetbrains-mono}` + `@fontsource/shippori-mincho` in main.tsx (NOT globals.css), so storybook also lacks them. Shipped via cfg.extraFonts. The static fallback names "DM Sans"/"JetBrains Mono" are aliased to the variable woff2 in `apps/web/.ds-entry/fonts-alias.css` so FONT_MISSING clears.
- AniList cover images (s4.anilist.co) are unreachable at sync time (no egress) — image-bearing components grade with gray cover placeholders on BOTH panels; covers load at view time on claude.ai. Grade structure/layout, not the missing remote images. Run compare with sandbox disabled regardless (other local assets).
- Card components (AnimeCard, etc.) render wider than storybook's fixed poster-width decorator when placed in the wide grid cell — a framing difference, graded `close`. Subagents may constrain via an owned preview if it materially helps.

## Wave learnings (folded from fan-out)

- [GENERAL] Locale: previews render Polish (i18n default), storybook renders English (DEFAULT_LANGUAGE='en'). Structure/styling identical → graded match. Both are real supported locales; not chased. To align, set i18n.changeLanguage(DEFAULT_LANGUAGE) in DesignProvider (cosmetic, costs a full rebuild + re-render; grades carry).
- [GENERAL] A few components leak raw i18n keys for namespaces not resolved in the preview (e.g. ActivityFeed: `newTab.recents.relative.months`, `activity.retry`) → graded close. Isolated; harness i18n-namespace gap.
- [GENERAL] Store-seeded story variants: stories that seed a Zustand store via `beforeEach`/decorators (not args) render the EMPTY/DEFAULT state in previews (the composeStories twin doesn't run beforeEach; meta decorator reset wins). Affected "filled/selected" variants graded close. Forcing the store in an owned preview works but drops the i18n `settings` namespace (raw keys) — mutually exclusive, so left as close. Components seen: BatchActionBar, BackgroundPanel, BackgroundSettings, BackgroundStep (more likely in later waves).
- [GENERAL] AppBackground (`--glow-1/--glow-2`, very low opacity) renders near-black in preview vs subtle glows in storybook → close, cosmetic full-bleed backdrop.
- [GENERAL] AccountsSection shows an extra "desktop-only" alert (web-vs-desktop env branch) — consistent, graded match.
- [GENERAL] Brand image assets (mascot PNG sprites, in-app screenshots) referenced by public/absolute path don't resolve in the preview card (they load in storybook). Affected: SplashHero, SplashScreen, MascotPreview, MascotSection, ErrorBoundary, SuiteSection — broken-image glyph in preview. Graded close; cosmetic asset-path limitation (preview cards don't serve the app's public/ dir).
- PORTAL/INTERACTION DIALOGS render nothing statically on BOTH panels (sb-error, "no storybook root content") — open dialog portals to document.body / needs play(). cardMode:single + primaryStory does NOT help. SKIPPED via cfg.overrides.<Name>.skip (all stories → floor card; component still ships bundle export + .d.ts + .prompt.md). - [GENERAL] `position:fixed; inset-0` full-window overlays (OnboardingWizard, SplashScreen) blank the PER-STORY compare capture (root bbox collapses to height:0) even though content renders fine and the cardMode:single product card is faithful — graded close, kept (not skipped). - AnimeInfoLinks/People/Recommendations: first story is empty-by-design; set cfg.overrides.<Name>.primaryStory to the populated story (WithStreaming/WithPeople/WithRecommendations) so the card + render-check use the populated render.
- [GENERAL] Store-seeded components that self-gate on store/electronAPI state seeded in `beforeEach` (the composeStories twin never runs beforeEach) render empty/null in preview → close, no fix. Many components (DiscordSection, MascotSection, SyncBadge, NotificationBell, SubscribeBellButton, AnimeDetailExtras, NewTabPage, and "filled/populated/selected" story variants across LibraryView, ProfileView, FeedView, DiscoverView, MalStatsPanel, etc.).
